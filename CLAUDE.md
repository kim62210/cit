# cit Project Instructions

## Language Policy

- All repository-authored content must be written in English.
- This rule overrides any broader default that would otherwise produce Korean project artifacts.
- Chat responses may follow the active assistant-level language policy, but anything written into the repository must be English.

## Commits and Git Artifacts

- All commit messages must be written in English.
- Use English Conventional Commit style prefixes such as `feat:`, `fix:`, `refactor:`, `docs:`, `test:`, and `chore:`.
- All branch names, PR titles, and PR descriptions for this repository must be written in English.

## Code and Documentation

- All source code comments, docstrings, README updates, design notes, test names, fixture descriptions, CLI help text, user-facing output, log messages, and error messages must be written in English.
- If you touch existing text, rewrite the touched text in English unless it is an external quote, proper noun, or user-provided content that must remain unchanged.

## Operational Guidance

- Before creating a commit, verify that the staged diff does not introduce Korean text into repository files unless it is intentionally preserved external content.
- If a higher-level instruction conflicts with this repository rule for project artifacts, follow this file for repository content and use English.
